# v0.3 GitHub Action Design

**Date:** 2026-06-15
**Phase:** v0.3 (per `docs/PRODUCT_PLAN.md`)
**Status:** Approved

## Goal

Add a Composite Action that detects drift in generated brief files (`AGENTS.md`, `docs/agent/*.md`) on every pull request. Fail the PR check if any committed brief file differs from freshly generated content.

## Scope (in)

- Composite Action in this repo (`.github/actions/repo-brief-check/action.yml`)
- Workflow in this repo consuming the action (`.github/workflows/brief-check.yml`)
- Make generated output deterministic by removing the in-file timestamp
- Fixture + test coverage for drift detection
- Docs updates (README, PRODUCT_PLAN)
- `0.3.0` release

## Scope (out)

- `verify` subcommand (rejected, YAGNI)
- Auto-fix / open patch PR (v0.5 Safe Fix Mode)
- MCP server (v0.4)
- Monorepo subdirectory scanning (YAGNI)
- Non-GitHub CI support

## Key Design Decisions

| Decision | Rationale |
|----------|-----------|
| Composite Action in same repo | Zero extra publishing burden; users reference via `uses: rogerdigital/repo-brief/.github/actions/repo-brief-check@<ref>` |
| Reuse published npm package via `npx` | Action behavior decoupled from this repo's source; no Docker build |
| Drift = real file content diff | Deterministic detection; catches real drift |
| Fail + report (gate), no auto-commit | Matches v0.3 scope; Safe Fix Mode deferred to v0.5 |
| Only check existing brief files | Missing files do not count as drift (don't false-alarm on un-adopted repos) |
| Trigger: `pull_request` only | Matches PRODUCT_PLAN wording; no main/push noise |
| Remove in-file timestamp | Required for deterministic diff; otherwise every run reports drift |

## Architecture

### Data Flow

```
PR opened/updated
   ↓
checkout code (includes committed AGENTS.md, etc.)
   ↓
npx @rogerdigital/repo-brief brief --cwd "$RUNNER_TEMP/gen"   ← generate into temp dir
   ↓
for each committed brief file: git diff --no-index <committed> <generated>
   ↓
any diff non-empty → list drift files + exit 1 (PR check fails)
all identical       → print "up to date" + exit 0
```

### Components

1. **`src/render.ts`** — remove `- Generated: ${brief.generatedAt}` line from `renderAgentsMd`
2. **`src/cli.ts`** — print `Generated at: <timestamp>` in scan summary log (keeps diagnostic value out of files)
3. **`.github/actions/repo-brief-check/action.yml`** — composite action
4. **`.github/workflows/brief-check.yml`** — this repo's consumer workflow
5. **`examples/fixtures/drift/`** — fixture with intentionally stale brief
6. **`tests/render.test.ts`**, **`tests/fixtures.test.ts`** — test updates
7. **README.md**, **docs/PRODUCT_PLAN.md** — docs updates

## Timestamp Handling (Determinism)

Current state (`render.ts:44-50`): `generatedAt` appears only in `AGENTS.md` Project Snapshot section. The other three files do not contain timestamps (verified).

Change:
- Keep `RepositoryBrief.generatedAt` field in `types.ts`; scanner still produces it
- Keep timestamp in CLI log output
- Remove the `- Generated: ${brief.generatedAt}` line from `renderAgentsMd`

Result: same input → byte-identical output, enabling clean diff.

Rejected alternative: normalize timestamp at diff time. Rejected because it adds diff logic complexity and is fragile (normalize rules must track render changes).

## Composite Action Structure

**Location:** `.github/actions/repo-brief-check/action.yml`

**Inputs:**
- `package-version` (default `latest`): npm version/tag of `@rogerdigital/repo-brief`

**Steps:**
1. `actions/setup-node@v4` (node 20)
2. `npx -y @rogerdigital/repo-brief@<version> brief --cwd "$RUNNER_TEMP/gen"` (generate into temp dir)
3. Drift detection loop (bash):
   - For each of `AGENTS.md`, `docs/agent/repo-map.md`, `docs/agent/commands.md`, `docs/agent/risk-zones.md`:
     - Skip if not present in workspace (missing ≠ drift)
     - `git diff --no-index --quiet <committed> <generated>`; if differs, record file and print full diff
   - If any drift files: print `::error::` annotation + list + `exit 1`
   - Else: print "up to date" + `exit 0`

**Permissions:** `contents: read` only (gate mode, no writes).

**Key behaviors:**
- `set -euo pipefail` — real errors surface, not masked by drift logic
- `git diff --no-index` — compares arbitrary paths, no shared git tree needed
- Does not call `actions/checkout` — caller workflow must checkout first (documented)
- No network beyond `npx` install

## Consumer Workflow

**Location:** `.github/workflows/brief-check.yml`

- Trigger: `pull_request` (all paths except the workflow itself)
- `permissions: contents: read`
- `concurrency` group with `cancel-in-progress: true`
- `timeout-minutes: 5`
- Steps: `actions/checkout@v4` → `uses: ./.github/actions/repo-brief-check`

Separate from existing `ci.yml` (which runs `pnpm verify`). The two run in parallel.

## Error Handling & Edge Cases

| Situation | Behavior | Exit Code |
|-----------|----------|-----------|
| `npx` install failure | step fails via `set -e` | ≠0 |
| Scan throws | repo-brief exits 1 | 1 |
| All committed briefs up-to-date | print success | 0 |
| Drift detected | `::error::` annotation + list + exit 1 | 1 |
| Some brief files missing | skip missing, check rest | depends on others |

Edge cases:
- PR edits brief but not regenerated → drift → fail (expected)
- PR edits source and brief consistently → pass if diff matches generated
- First-time repo with no briefs → all skipped → pass (no false alarm)
- Monorepo `--cwd` subdirectory → not supported in v0.3 (YAGNI); future `working-directory:` input

## Release Flow

1. Implementation complete + all tests pass
2. Local manual acceptance checklist green
3. `package.json` bump to `0.3.0`
4. `pnpm publish` (prepack auto-builds)
5. Git tag `v0.3.0` (external users reference `@v0.3` series tag)
6. GitHub release notes
7. Merge PR to main

**Critical ordering:** npm package MUST publish before git tag — otherwise external users referencing `@v0.3` get an `npx` lookup failure.

## Risks

| Risk | Probability | Impact | Mitigation |
|------|-------------|--------|------------|
| `npx` slow/fail on runner | Low | Flaky red PR check | `timeout-minutes: 5`; GitHub runner network stable |
| Future render adds non-deterministic field | Medium | False drift reports | Determinism test gates CI |
| External user references `@v0.3` before npm publish | Low | Action errors | Release docs stress ordering |
| `brief-check.yml` triggers spuriously | Low | Extra checks | `paths` excludes self workflow |
